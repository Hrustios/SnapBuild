# Сервис мониторинга конкурентов SnapBuild

## Описание проекта

Цель проекта — разработка автономного сервиса мониторинга конкурентов SnapBuild с автоматическим отслеживанием изменений на их сайтах, страницах релизов, блогах и других источниках информации.

Сервис предназначен для:

- мониторинга обновлений конкурентов;
- отслеживания появления новых функций;
- отслеживания изменений тарифов и возможностей сервисов;
- хранения истории изменений.

Работа сервиса должна выполняться автоматически без участия команды.

# Архитектура системы

Сервис состоит из следующих компонентов.

## 1. Модуль парсинга

Функции:

- загрузка списка URL из конфигурационного файла YAML;
- получение HTML-страниц;
- извлечение полезного текстового содержимого;
- создание снапшотов страниц;
- сохранение данных.

Используемые технологии:

- Python
- BeautifulSoup
- Requests
- Playwright (для сайтов с динамической загрузкой контента через JavaScript)

---

## 2. Хранилище данных

Для хранения используется SQLite.

База данных хранит:

- название конкурента;
- URL страницы;
- текстовый снапшот страницы;
- дату и время проверки.

Это позволяет:

- сравнивать версии страниц;
- фиксировать изменения;
- хранить историю обновлений.

---

## 3. Детектор изменений

Система сравнивает новый снапшот с предыдущим и определяет наличие изменений.

Фиксируются:

- новые функции;
- изменения текстов;
- изменение тарифов;
- обновления описания продукта;
- новые релизы.

Перед сравнением HTML очищается от лишних элементов, чтобы избежать ложных срабатываний.

# Конфигурация

Список конкурентов и URL хранится в YAML-файле.

Пример структуры:

```yaml
competitors:
  - name: "Lovable"
    urls:
      - "https://lovable.dev"
      - "https://lovable.dev/blog"
      - "https://docs.lovable.dev/changelog"

  - name: "Replit"
    urls:
      - "https://replit.com"
      - "https://blog.replit.com"
      - "https://docs.replit.com/updates/"
```

# Обработка ошибок

Сервис предусматривает базовую обработку ошибок:

- сайт временно недоступен;
- ошибка подключения;
- превышение времени ожидания;
- ошибка парсинга страницы.

Ошибки логируются и не останавливают выполнение программы.
